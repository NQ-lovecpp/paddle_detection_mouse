**切入点：** 当老师问“你在项目中遇到过什么困难？”或“介绍一段你的实习经历”时。

#### 1. 背景与技术决策 (Situation & Decision) —— _这里体现你的工程视野_

> “我在**上海交大网络信息中心**实习期间，主要负责基于 **PaddlePaddle** 框架搭建一个实验小白鼠的图像分类模型。
> 
> 当时我面临一个棘手的基础设施问题：服务器运行的是 **Ubuntu 18.04**，其系统自带的 Python 版本（3.6）较老，无法满足新版 PaddlePaddle 和相关 AI 库的依赖需求。
> 
> 我当时评估了两种方案： 第一种是源码编译安装（Source Compile）新版 Python。但我考虑到这会带来后续的包管理混乱，且极易引发‘依赖地狱’，不利于环境的长期维护。 第二种是升级操作系统到 **Ubuntu 20.04** ，因为 20.04 原生支持较新的 Python 版本，能从系统层面解决依赖问题。
> 
> **为了保证环境的稳定性和可维护性，我果断选择了升级操作系统。**”

#### 2. 突发故障 (Complication) —— _引出危机_

> “然而，升级过程虽然顺利，但重启后服务器却**彻底失联**了——SSH 无法连接，Ping 不通。 更糟糕的是，由于是从 18.04 跨版本升级，原有的网络配置工具 `Netplan` 和 `NetworkManager` 在升级中发生了组件更迭或失效 ，导致我无法用熟悉的工具恢复网络。”

#### 3. 底层排查 (Action) —— _展示你的硬核技术_

> “我意识到必须进行底层排查，于是我接上显示器直连服务器，采取了**分层排查策略**：
> 
> * **第一步（服务层）：** 我检查了 `sshd` 进程，发现它活着 ，排除了服务崩溃的可能。 * **第二步（配置层）：** 我发现新系统启用了 `systemd-networkd` ，于是尝试将旧的静态 IP（10.110.1.62）迁移过去，但网络依然显示 `NO-CARRIER`（无载波）。
> 
> - **第三步（物理链路层）：** 这是破局的关键。我没有死磕配置文件，而是用 `ethtool` 命令检查物理接口状态。 我发现配置文件中原本指向的网卡 `ens8f0`，其 `Link detected` 状态竟然是 **NO** ；而另一个看似无关的接口 `ens8f1` 状态却是 **YES** 。
>     
> 
> **我立刻通过这个现象定位了根因：** 操作系统升级导致内核对 PCIe 设备的枚举顺序或命名规则发生了‘漂移’，导致原本的配置文件‘指鹿为马’，绑定到了错误的物理网口上 。”

#### 4. 解决与沉淀 (Result) —— _展示成果与文档能力_

> “定位问题后，我修改了 `systemd-networkd` 的配置文件，将 IP 绑定到正确的 `ens8f1` 接口上 ，重启服务后网络瞬间恢复 。
> 
> 这次修复不仅让我顺利完成了后续 PaddlePaddle 环境的搭建，我还以此为案例，梳理了从 **Kernel -> Systemd -> Networkd** 的网络栈架构图 ，并输出了一份技术文档分享给团队。这也让我深刻理解了：做 AI 工程，不能只懂上层算法，底层系统的掌控力同样关键。”

### 面试官可能针对“Python决策”的追问（请准备好）

**Q: 为什么不使用 Conda 或者 Docker 来解决 Python 版本问题？**

- **回答策略（显出你的实战经验）：**
    
    - “老师您说的很对，通常我们会用 Conda 或 Docker。但在当时的特定环境下，这台服务器需要直接调用底层的物理 GPU 驱动，且作为一台既有的物理机，还需要承载一些其他系统服务。
        
    - 当时的显卡驱动（NVIDIA Driver）与旧内核在 Ubuntu 18.04 下存在一些兼容性隐患。
        
    - 所以综合考虑，升级 OS 是一次**‘还技术债’**的操作，不仅解决了 Python 版本，也更新了 Kernel 和驱动支持，为后续的 GPU 训练打好更稳固的基础。”
        

### 这个新版本的亮点在哪里？

1. **拒绝“学生气”**：很多学生遇到问题只会 Google 怎么装软件。你通过对比“源码编译”和“系统升级”的优劣，展示了**架构师思维（Architectural Thinking）**。
    
2. **逻辑闭环**：如果不升级 -> Python环境乱 -> 难以维护；升级了 -> 遇到网络坑 -> 填坑成功。这个逻辑链条非常完美。
    
3. **理由充分**：强调“为了避免不便”听起来像偷懒，但强调**“为了环境的可维护性和标准化”**，就是专业的体现。
